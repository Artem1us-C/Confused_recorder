# Confused Recorder

Легковесная десктопная утилита для записи экрана в реальном времени с функцией мгновенного размытия (blur) конфиденциальных данных. Инструмент разработан для оптимизации работы QA-инженеров, технических писателей и специалистов службы поддержки, которым необходимо записывать баг-репорты или видеоинструкции, исключая попадание в кадр персональных данных, токенов, паролей или коммерческой тайны.

Главная особенность — **Zero Post-Production**. Замазывание областей происходит прямо во время захвата экрана. Финальный mkv-файл со звуком готов к отправке  в ту же секунду, когда нажата кнопка «Стоп» (без долгого рендеринга и конвертации).

---


* **Защита данных «на лету»**: Размытие всего экрана с возможностью динамического выделения «чистых» зон («протирка дырок») прямо во время записи.
* **Кастомизация параметров**: Изменение степени размытия (Blur Strength) и цвета маскирующей дымки в реальном времени.
* **Запись аудио**: Поддержка захвата звука с микрофона через системное аудио-API.
* **Оптимизация производительности**: Низкая нагрузка на CPU и RAM за счет масштабирования кадра перед обработкой и прямой передачи байт.
* **Автономность**: Локальное сохранение настроек (выбранная папка, аудиоустройство) в компактный конфигурационный файл.

---



Проект спроектирован с упором на максимальную отзывчивость интерфейса при высокой интенсивности дискового ввода-вывода.

* **UI Фреймворк**: `Avalonia UI` (.NET 8) — кроссплатформенный графический движок. Логика интерфейса полностью отвязана от потока видеозахвата, что исключает дедлоки и падения UI по таймаутам рендеринга Windows (`RequestCommitAsync`).
* **Обработка изображений**: `OpenCvSharp4` (OpenCV wrapper для .NET) — используется для высокоскоростного захвата кадров экрана (`CopyFromScreen`), работы с матрицами (`Mat`), изменения размера кадра (Resize) и наложения фильтра Гаусса (`GaussianBlur`).
* **Аудио-подсистема**: `NAudio` (WaveInEvent) — безопасный сбор аудиоданных без использования нестабильных COM/WASAPI-объектов в интерфейсном потоке.
* **Кодирование видео**: `FFmpeg` — видео и аудио упаковываются «на лету». Вместо создания тяжелых промежуточных файлов, сырые BGR-байты из памяти C# и аудиопоток транслируются напрямую в анонимный пайп (`StandardInput.BaseStream`) процесса `ffmpeg.exe` с параметрами `-preset ultrafast` и `-shortest`.
